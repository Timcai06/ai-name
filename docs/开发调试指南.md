# 开发调试指南 — AI 智能取名系统

> 怎么在不写前端代码的情况下，直接测试后端接口

---

## 一、为什么用 Swagger

开发时前端还没写完，或者想快速验证后端某个接口对不对。Swagger 让开发者**在网页上直接调接口、看返回结果**，不用 curl 命令行，不用 Postman。

**地址**：`http://localhost:8000/docs`（后端必须启动）

---

## 二、核心概念

### Bearer Token 是什么

Token 就是**临时身份证**。登录成功后后端发一张"票"，后续每个请求带着这张票，后端就知道你是谁。

```
登录 → 后端返回 token（一张票）
     → 浏览器存起来
     → 之后每个请求自动带上
     → 后端验票，放行
```

### 为什么要贴 token

打开 Swagger 的时候，它不知道你是谁。你必须告诉它"我是 admin，这是我的票"。

主流开发框架（FastAPI、Spring、Express）全是这个模式——Bearer Token 认证。不是什么生僻方法，是行业标准。

## 三、Swagger 操作步骤（带截图说明）

### 第 1 步：打开 Swagger

浏览器访问 `http://localhost:8000/docs`

你会看到所有接口按分组排列：
- **auth**：注册、登录、登出、用户信息、日志
- **naming**：生成名字、微调名字、历史列表、历史详情
- 最下面还有个 **health**：健康检查

### 第 2 步：登录获取 token

1. 找到 **auth** 分组，点 `POST /api/auth/login`
2. 点右侧 **Try it out** 按钮（蓝色）
3. 在 Request body 输入框里，把默认的 JSON 改成你的账号信息：

```json
{
  "username": "admin",
  "password": "246809cyk"
}
```

4. 点下面蓝色的 **Execute** 按钮
5. 往下滑看 **Server response** 区域的 **Response body**：

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxIiwiZXhwIjoxNzgzMTY0NjA1fQ.-Wr_NF8MQxbYhXGtKJcGPJhxpPZssN4PkMWKGkQfY1Y",
  "username": "admin"
}
```

`token` 后面的那一长串 `eyJ...` 就是你的票。**复制整段**。

### 第 3 步：在 Swagger 中"登录"（Authorize）

1. 点页面右上角的绿色 **Authorize** 按钮（旁边有个锁的图标）
2. 弹窗里有一个输入框，标签是 **Value**
3. 把你刚才复制的 token 粘贴进去
4. 点 **Authorize** → 关闭弹窗

现在 Swagger "知道你登录了"。所有带锁图标的接口都可以调了。

### 第 4 步：调用需要登录的接口

以查看日志为例：

1. 找到 **auth** → `GET /api/auth/logs`（旁边有个锁图标，说明需要登录）
2. 点 **Try it out**
3. 直接点 **Execute**（不需要改任何参数）
4. 下方 **Server response** 显示结果：

```json
{
  "total": 5,
  "items": [
    {
      "id": 1,
      "username": "admin",
      "action": "register",
      "ip_address": "127.0.0.1",
      "created_at": "2026-07-03T11:14:15"
    },
    {
      "id": 2,
      "username": "alice",
      "action": "login",
      "ip_address": "127.0.0.1",
      "created_at": "2026-07-03T11:15:22"
    }
  ]
}
```

每条记录说明：

| 字段 | 含义 |
|------|------|
| username | 谁 |
| action | 做了什么（register/login/logout） |
| ip_address | 从哪个 IP 来的 |
| created_at | 什么时候 |

### 第 5 步：测试生成名字

1. 找到 **naming** → `POST /api/naming/generate`
2. **Try it out**
3. Request body 填：

```json
{
  "surname": "李",
  "gender": "female",
  "style": "温婉典雅",
  "expectations": "希望名字有诗意，出自古诗词"
}
```

4. **Execute**
5. 等几秒，返回：

```json
{
  "conversation_id": "a1b2c3d4-...",
  "names": [
    {
      "full_name": "李清婉",
      "meaning": "清丽温婉，如诗如画...",
      "wuxing": "水木",
      "source": "《诗经·秦风》"
    }
  ]
}
```

## 四、不同角色看不同的东西

| 接口 | 未登录 | 普通用户 | 管理员(admin) |
|------|--------|----------|---------------|
| POST /auth/register | ✅ | ✅ | ✅ |
| POST /auth/login | ✅ | ✅ | ✅ |
| POST /naming/generate | 401 | ✅ | ✅ |
| POST /naming/refine | 401 | ✅ | ✅ |
| GET /naming/history | 401 | ✅（只看自己） | ✅（只看自己） |
| GET /auth/logs | 401 | 403 | ✅（看所有人） |

- **401**：没登录，没贴 token
- **403**：登录了，但权限不够
- **200**：正常

## 五、常见问题

### Q：token 过期了怎么办？

A：重新登录拿新的。默认有效期 24 小时。过期后接口返回 `登录已过期，请重新登录`。

### Q：为什么不用输入密码就能看日志？

A：因为你已经贴了 token。token 就是密码的替代品——登录一次，之后靠 token。前端网页也是这么做的：登录后存 token 在浏览器里，之后自动带。

### Q：能不能不用 token，直接在 URL 里传密码？

A：绝对不行。URL 里的内容会被记录在浏览器历史、服务器日志里。Bearer Token 模式是行业标准，安全得多。

### Q：能加一个管理后台页面直接在网站上看日志吗？

A：能。前端加一个 AdminPanel 页面，admin 登录后在网站上直接看。项目从难度 2 升级时加。

## 六、标准开发调试流程

```
1. 终端 1：cd backend && uvicorn app.main:app --reload
2. 终端 2：cd frontend && npm run dev
3. 浏览器窗口 1：localhost:5173（前端，正常使用）
4. 浏览器窗口 2：localhost:8000/docs（Swagger，调试后端）
```

前端写好之前或出 bug 时，直接在 Swagger 独立调后端接口，快速定位问题在哪个环节。

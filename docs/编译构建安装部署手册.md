# 编译构建安装部署手册 — AI 智能取名系统

> 版本：2.0 | 日期：2026-07-03

---

## 1. 环境要求

| 软件 | 最低版本 | 说明 |
|------|----------|------|
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端构建环境 |
| npm | 9+ | 包管理器 |
| DeepSeek API Key | — | 从 https://platform.deepseek.com 获取 |

---

## 2. 获取源码

```bash
git clone <仓库地址>
cd ai-naming
```

或解压 `src.zip` 到目标目录。

---

## 3. 后端部署

### 3.1 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

`requirements.txt` 包含：
- fastapi, uvicorn — Web 框架
- httpx — HTTP 客户端
- pydantic, python-dotenv — 数据校验与配置
- sqlalchemy, pymysql — 数据库 ORM
- python-jose[cryptography] — JWT 认证
- passlib[bcrypt], bcrypt — 密码哈希

### 3.2 配置环境变量

创建 `backend/.env` 文件：

```ini
DEEPSEEK_API_KEY=sk-your-key-here
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_admin_password
DATABASE_URL=sqlite:///./naming.db
```

| 变量 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| DEEPSEEK_API_KEY | 是 | — | DeepSeek API 密钥 |
| ADMIN_USERNAME | 否 | admin | 管理员用户名 |
| ADMIN_PASSWORD | 否 | admin123456 | 管理员密码 |
| DATABASE_URL | 否 | sqlite:///./naming.db | 数据库连接 |
| SECRET_KEY | 否 | (内置默认) | JWT 签名密钥，**生产环境必须修改** |

### 3.3 切换 MySQL

设置 `DATABASE_URL`：

```ini
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/naming_db
```

### 3.4 启动服务

```bash
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### 3.5 验证

```bash
curl http://localhost:8000/api/health
# {"status":"ok","service":"ai-naming"}
```

浏览器打开 `http://localhost:8000/docs` 查看 API 文档。

---

## 4. 前端部署

### 4.1 安装依赖

```bash
cd frontend
npm install
```

### 4.2 开发模式

```bash
npm run dev
```

浏览器打开 `http://localhost:5173`。Vite 自动将 `/api` 请求代理到后端 8000 端口。

### 4.3 生产构建

```bash
npm run build
```

构建产物在 `frontend/dist/`。

---

## 5. 一体化部署

### 5.1 Nginx 配置

```nginx
server {
    listen 80;
    server_name _;

    location / {
        root /path/to/frontend/dist;
        try_files $uri /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 5.2 后端进程管理（Linux）

```ini
# /etc/systemd/system/ai-naming.service
[Unit]
Description=AI Naming Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/ai-naming/backend
Environment=DEEPSEEK_API_KEY=sk-xxx
Environment=SECRET_KEY=production-secret-key
ExecStart=/usr/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

## 6. 故障排查

| 问题 | 解决 |
|------|------|
| ImportError: jose | `pip install python-jose[cryptography]` |
| ImportError: passlib | `pip install passlib[bcrypt]` |
| ImportError: sqlalchemy | `pip install sqlalchemy` |
| DEEPSEEK_API_KEY 未设置 | 创建 `backend/.env` 或设置环境变量 |
| 前端 404 | 后端未启动，检查 uvicorn |
| 前端 CORS 错误 | Vite proxy 未生效，检查 `vite.config.ts` |
| AI 返回 504 | DeepSeek API 超时，检查网络 |
| 登录后 token 401 | token 已过期（24h），重新登录 |
